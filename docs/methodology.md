# Methodology / 方法说明

## 方法来源

本项目的方法灵感，来自斯坦福 Life Design / Designing Your Life 体系中的 **Odyssey Planning** 思路。该方法鼓励用户围绕未来若干年，构想多条不同的人生路径，用来探索可能性、比较取舍，并通过小规模行动进行验证，而不是把人生理解成只有一种标准答案。 

## 本项目做了什么

在此基础上，本项目做的是一种 **AI 工具化整理与结构化实现**，主要包括：

- 将人生方向思考拆成可提问、可推演、可比较的流程
- 用 A / B / C 多路径方式帮助用户看到不同取舍逻辑
- 用“普通一天模拟”“横向比较”“7天 / 30天验证动作”等结构，增强现实感和行动感
- 强调低成本验证，而不是命运预测或替用户做决定

## 独立性说明

本仓库是一个独立项目。  
它**受相关方法启发**，但**不是**斯坦福官方产品、官方课程工具、认证项目或附属服务，也**不代表**官方立场。 

## 边界说明

本项目用于人生方向模拟与决策辅助，帮助用户把模糊的人生选择，转化为更清晰、可比较、可验证的路径。  
它不用于命运预测，也不替代心理、医疗、法律、财务等专业意见。

# Methodology

这个项目参考的是一种“多路径人生推演”思路。

核心不是预测未来，而是通过构建多个可能的人生版本，帮助用户：

1. 更清楚地理解自己
2. 看见不同选择背后的代价与吸引力
3. 把模糊纠结转化为可比较的路径
4. 用低成本实验验证，而不是只在脑内空转

## 核心流程

1. 澄清问题
2. 收集关键信息
3. 提炼价值观与约束
4. 生成 3 条路径
5. 展开生活场景模拟
6. 横向比较
7. 设计验证动作

## 为什么不是“预测”

人生不是股票走势图。
这个项目不试图给出确定答案，而是帮助用户把可能性看得更清楚。
